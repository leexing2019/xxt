# 添加药品页面优化设计文档

**日期**: 2026-03-21
**状态**: 待审查

---

## 1. 概述

### 1.1 项目背景
当前添加药品页面逻辑过于复杂，不适合老年人使用。需要重构为简单清晰的三步走流程，并集成专业 AI API 提升识别准确率。

### 1.2 设计目标
- 简化操作流程，适合老年人独立使用
- 统一所有添加方式的交互流程
- 集成百度 AI API 提升语音和图像识别准确率
- 创建管理后台，支持用药计划管理和 API 配置

---

## 2. 适老化添加药品流程

### 2.1 三步走流程

```
┌─────────────────────────────────────────────────────────────┐
│  步骤 1: 选择药品  →  步骤 2: 设置时间  →  步骤 3: 确认      │
└─────────────────────────────────────────────────────────────┘
```

#### 步骤 1: 选择药品
**功能入口**（统一入口，多种选择方式）:
- 常用药品列表（50 种高频药品，支持分类筛选）
- 搜索框（支持拼音首字母、同音字、别名）
- 语音输入（按住说话，自动识别药品名）
- 拍照识别（拍摄药盒，AI 识别药品信息）

**界面要点**:
- 大字体（最小 18px）
- 高对比度配色
- 药品卡片显示：药名 + 外观缩略图 + 用法简述

#### 步骤 2: 设置时间
**表单字段**:
- 服用时间（时间选择器，支持多个时间点）
- 用量（数字选择器 + 单位）
- 用药频率（每日/每周选择）
- 用药周期（开始日期、结束日期）

**界面要点**:
- 大按钮时间选择器
- 默认值预设（如每日 1 次、每次 1 片）
- 实时预览当日服药时间

#### 步骤 3: 确认
**显示内容**:
- 药品名称（大字体突出）
- 服用时间列表
- 用量说明
- 其他信息折叠（外观描述、注意事项等可展开查看）

**操作按钮**:
- 确认添加（主按钮，大尺寸）
- 返回修改（次按钮）
- 取消（小按钮）

### 2.2 导航设计

**多处入口**:
- 首页顶部"添加药品"按钮
- 用药列表页底部"添加计划"按钮
- 底部 Tab 导航"添加"按钮（可选）

**返回机制**:
- 每步都有返回按钮
- 支持上一步/下一步导航
- 允许随时取消退出

**完成行为**:
- 完成后停留在添加页
- 显示"添加成功"提示
- 自动清空表单，可继续添加下一个

### 2.3 适老化视觉规范

| 元素 | 规格 |
|------|------|
| 正文字体 | 18px 最小，标题 24px+ |
| 按钮高度 | 最小 48px |
| 主色调 | 高对比度蓝色 (#1976D2) |
| 成功色 | 绿色 (#4CAF50) |
| 错误色 | 红色 (#F44336) |
| 行间距 | 1.6 倍行高 |
| 点击区域 | 最小 44x44px |

### 2.4 语音引导

- 步骤切换时语音播报当前步骤
- 操作成功/失败语音反馈
- 关键操作二次确认

## 3. API 集成方案

### 3.1 百度语音识别 API

**用途**: 语音输入药品名称

**配置项**:
- `BAIDU_APP_ID`
- `BAIDU_API_KEY`
- `BAIDU_SECRET_KEY`

**接口**:
```
POST https://vop.baidu.com/server_api
```

**流程**:
1. 获取 access_token
2. 录制语音（WAV 格式，16k 采样率）
3. 发送语音数据
4. 解析识别结果
5. 匹配本地药品库

### 3.2 百度药品图片识别 API

**用途**: 拍照识别药盒上的药品信息

**配置项**: 同上（使用相同认证）

**流程**:
1. 拍照/上传图片
2. 调用 OCR/药品识别接口
3. 解析返回的药品名称、规格等信息
4. 自动填充表单
5. 用户确认/修改

### 3.3 API 配置管理

**方案**: 后台可配置，前端读取

- 管理后台提供 API 配置页面
- 配置存储于数据库 `app_settings` 表
- 前端启动时读取配置
- 支持动态更新，无需重新部署

## 4. 管理后台设计

### 4.1 技术栈
- 独立 admin 路由（/admin/*）
- 与前端共享 Supabase 认证
- 需要管理员权限

### 4.2 功能模块

#### 4.2.1 用药计划管理
**功能**: 增删改查用户用药计划

**列表页面**:
- 用户筛选
- 药品名称搜索
- 状态筛选（活跃/停用）
- 分页展示

**详情/编辑**:
- 查看完整用药计划信息
- 编辑时间、用量、频率
- 停用/启用计划
- 删除计划

#### 4.2.2 用药统计报告
**功能**: 数据可视化展示

**统计维度**:
- 总用户数、活跃用户数
- 总用药计划数
- 今日/本周/本月服药记录数
- 服药依从性统计
- 热门药品排行

**图表类型**:
- 折线图（依从性趋势）
- 饼图（药品分类占比）
- 柱状图（每日服药次数）

#### 4.2.3 API 配置页面
**功能**: 管理第三方 API 配置

**配置项**:
- 百度语音识别 API 密钥
- 百度图片识别 API 密钥
- 其他扩展 API 配置

**操作**:
- 新增/编辑配置
- 测试连接
- 保存生效

### 4.3 数据库设计

#### admin_users 表
```sql
CREATE TABLE admin_users (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  email TEXT UNIQUE NOT NULL,
  password_hash TEXT NOT NULL,
  role TEXT DEFAULT 'admin',
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);
```

#### app_settings 表
```sql
CREATE TABLE app_settings (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  key TEXT UNIQUE NOT NULL,
  value JSONB NOT NULL,
  description TEXT,
  updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);
```

**新增服务**: `src/services/app-config.ts`

```typescript
// 从后台读取配置
export async function fetchAppConfig() {
  const { data } = await supabase
    .from('app_settings')
    .select('key, value')
    .in('key', ['baidu_app_id', 'baidu_api_key', 'baidu_secret_key'])

  return data?.reduce((acc, item) => ({
    ...acc,
    [item.key]: item.value
  }), {})
}
```

## 6. 错误处理

### 6.1 API 调用失败
- 语音识别失败：降级到手动输入
- 图片识别失败：显示识别结果编辑框
- 网络错误：提示检查网络连接

### 6.2 数据验证
- 药品名称必填
- 服用时间至少一个
- 用量必填
- 结束日期不能早于开始日期

### 6.3 用户反馈
- Toast 提示操作结果
- 表单错误实时显示
- 关键操作二次确认
