# 漏服检测逻辑设计文档

> **创建日期**: 2026-03-26
> **状态**: 已完成
> **涉及页面**: 首页 (index.vue)

## 概述

实现漏服自动检测逻辑，基于动态 24 小时窗口和 4 小时阈值判定，简化首页展示同时避免鼓励用户补服。

---

## 1. 核心逻辑

### 1.1 24 小时动态窗口

**设计原则**：首页只显示当前时间点后 24 小时内的服药计划。

**判定逻辑**：
```typescript
const now = new Date()
const currentMinutes = currentHour * 60 + now.getMinutes() // 当前时间的分钟数（0-1439）
const scheduledMinutes = scheduleHour * 60 + scheduleMinute // 计划时间的分钟数

// 计算时间差（分钟）
let timeDiff = currentMinutes - scheduledMinutes

// 处理跨天情况
if (timeDiff < 0) {
  timeDiff += 24 * 60 // 加上 24 小时的分钟数
}

// 24 小时窗口判定
const isInWindow = timeDiff < 24 * 60
```

**效果**：
- 周一 08:00 的计划，在周二 08:00 后自动从首页消失
- 如果用户没有服用，系统判定为漏服
- 新周期计划出现时，旧计划自动消失

### 1.2 4 小时漏服阈值

**设计原则**：超过计划时间 4 小时（240 分钟）未服用，判定为漏服。

**判定逻辑**：
```typescript
// 漏服判定：超过 4 小时（240 分钟）未服用
const isMissed = !log && timeDiff > 240
```

**过滤逻辑**：
```typescript
.filter(item => {
  // 过滤条件：
  // 1. 必须在 24 小时窗口内
  // 2. 如果已漏服，也过滤掉（不再展示）
  return item.isInWindow && !item.isMissed
})
```

**效果**：
- 漏服药物不再在首页展示
- 不鼓励用户补服（避免打乱后续用药节奏）
- 用户仍可在药品详情页查看历史记录

## 2. 状态判定时间表

| 时间范围 | 状态 | 是否展示 | 操作引导 |
|----------|------|----------|----------|
| 计划时间 < 15 分钟 | 提前 | 是 | 正常服用 |
| 计划时间 ±15 分钟 | 准点 | 是 | 正常服用 |
| 计划时间 +15 分钟 ~ 4 小时 | 过时 | 是 | 提醒后服用 |
| 计划时间 > 4 小时 | 漏服 | 否 | 不展示，不鼓励补服 |
| 第二天同一时间后 | 过期 | 否 | 自动消失 |

## 3. 数据结构

### 3.1 todayMedications 返回值

```typescript
interface TodayMedication {
  ...schedule,
  taken: boolean           // 是否已服用
  taken_time?: string      // 服用时间
  isMissed: boolean        // 是否已漏服
  isInWindow: boolean      // 是否在 24 小时窗口内
  timeDiff: number         // 距离计划时间的分钟差
}
```

### 3.2 计算属性

```typescript
const todayMedications: ComputedRef<TodayMedication[]>
const pendingMedications: ComputedRef<TodayMedication[]>  // 未服用（过滤漏服）
const takenMedications: ComputedRef<TodayMedication[]>    // 已服用
```

## 4. 边界情况处理

### 4.1 跨天计划

**问题**：如果计划时间是周一 08:00，当前时间是周二 07:00，如何计算？

**解决**：
```typescript
if (timeDiff < 0) {
  timeDiff += 24 * 60 // 加上 24 小时
}
```

### 4.2 星期切换

**问题**：周末与周一的切换如何处理？

**解决**：使用 `weekday` 过滤器，只显示当天星期的计划。

### 4.3 多次刷新

**问题**：页面刷新后数据重复加载？

**解决**：在 `onShow` 中检查用户是否切换，只在新用户登录时重新加载。

## 5. 用户体验优化

### 5.1 不鼓励补服

**设计理念**：
- 老年人记忆性差，补服可能导致重复用药
- 漏服后补服可能打乱后续用药节奏
- 4 小时阈值是医学上常用的安全窗口

### 5.2 友好提示

**漏服场景**：
- 不在首页展示漏服药物（减少焦虑）
- 可在详情页查看历史漏服记录
- 统计数据中计入漏服数量
